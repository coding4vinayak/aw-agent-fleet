The repository https://github.com/nullclaw/nullclaw
 is a framework for building autonomous AI agents. It is designed to run extremely lightweight AI assistants with almost no system resources.

Key facts about it:

~1 MB binary

Near-zero dependencies

Works on Raspberry Pi, servers, or cloud VMs

Supports 50+ AI providers (OpenAI, Gemini, DeepSeek, etc.)

Can connect to Telegram, Discord, WhatsApp, Slack, Email, etc.

Includes built-in tools like shell access, scheduling, memory, and web requests.

So this project is basically a tiny infrastructure to run AI agents anywhere.

Can i deploy this on vercel

Short answer: No — not properly.

You generally cannot deploy this type of project on Vercel.

Reason: Vercel is designed for serverless web apps, not long-running AI agents.

Why It Doesn't Fit Vercel

The Nullclaw agent behaves like a persistent background process.

It needs to:

run continuously

listen to events (Telegram, Slack, WhatsApp)

maintain memory/state

execute tasks over time

run system commands

Vercel functions are:

stateless

short-lived

limited to ~10–60 seconds execution

no background workers

So the process would terminate immediately.
